# 数据源优化计划

## 任务目标
对股票分析系统的数据源进行回测验证，移除不可靠的数据源，并支持新的优质数据源接入。

---

## 当前数据源现状

### 现有数据源（ultimate_data_fetcher.py）

| 数据源 | 用途 | 依赖 |
|--------|------|------|
| TickFlow | 实时行情、历史K线 | TICKFLOW_API_KEY |
| baostock | 财务数据、估值信息 | 无 |
| Tavily | 网络搜索补充财务数据 | TAVILY_API_KEY |
| XueqiuDataFetcher | 雪球数据 | 无 |
| AkshareDataFetcher | akshare数据 | 无 |

### 已知问题
- 从日志看到 `efinance` (Akshare内部) 存在连接失败问题
- 部分数据源可能返回不准确或缺失数据

---

## 计划任务

### 阶段一：数据源回测验证

1. **创建数据源测试脚本**
   - 针对每只股票，单独测试各数据源返回的数据
   - 对比关键指标（PE、PB、ROE、市值等）
   - 记录每个数据源的错误率、数据完整率

2. **定义数据质量标准**
   - PE: 0-1000（超出为异常）
   - PB: 0-100（超出为异常）
   - ROE: 0-100%（超出为异常）
   - 市值: > 1亿（低于为异常）
   - 数据完整度: 关键字段非空率 > 80%

3. **执行回测**
   - 使用5只以上股票进行回测
   - 记录每个数据源的成功率、准确性
   - 生成数据源质量报告

### 阶段二：数据源优化

1. **识别并移除问题数据源**
   - 错误率 > 30% 的数据源标记为不可用
   - 返回数据与真实值偏差 > 50% 的数据源移除
   - 记录移除的数据源及其问题原因

2. **保留可靠数据源**
   - 优先使用高成功率数据源
   - 保留备用数据源作为降级方案

### 阶段三：新数据源接入

1. **提供数据源接入接口**
   - 创建标准化的数据源适配器接口
   - 支持快速接入新的API

2. **用户可提供的数据源类型**
   - 股票基本信息API
   - 财务数据API
   - 历史行情API
   - 实时行情API

---

## 数据质量验证维度

| 指标 | 验证方法 | 合格标准 |
|------|----------|----------|
| 数据完整性 | 关键字段非空率 | > 80% |
| 数据准确性 | 与已知准确数据对比 | 偏差 < 10% |
| 数据时效性 | 数据更新时间 | < 24小时 |
| API可用性 | 请求成功率 | > 90% |
| 响应速度 | API响应时间 | < 5秒 |

---

## 预期成果

1. 生成数据源质量评估报告
2. 移除不可靠的数据源
3. 保留高可靠性的数据源
4. 建立数据源接入标准文档
5. 支持用户自定义数据源接入

---

## 风险与注意事项

1. **数据源过度集中风险**: 不要完全依赖单一数据源
2. **API限流风险**: 添加请求间隔和重试机制
3. **数据延迟风险**: 记录数据时间戳，标记过期数据
4. **向后兼容**: 确保移除数据源后不影响现有功能

---

## 用户支持

如需提供新的数据源API，请提供以下信息：
1. API地址和文档
2. 认证方式（API Key/OAuth/无认证）
3. 支持的数据类型（基本信息/财务数据/行情数据）
4. 请求频率限制
